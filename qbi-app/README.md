# QBI — ИИ-эксперт по финансовому управлению

Веб-приложение для предпринимателей: диалоговая диагностика финансового управления бизнеса на базе ИИ (Claude API), библиотека знаний и витрина услуг QBI Analytics.

## Что делает приложение

- **Эксперт QBI** (ядро продукта) — диалог с ИИ, обученным на методологии Максата Калибекова. Эксперт ведёт пользователя по 4 зонам (доходы → расходы → оборотный капитал → система управления), подсвечивает каждый этап, комментирует ответы и в финале выдаёт заключение с лепестковой диаграммой и рекомендацией продукта QBI. Собранный профиль лида предназначен для передачи в CRM.
- **Библиотека знаний** — карточки-свайпы: 60 цитат о финансах, глоссарий (20 терминов), «Бухгалтер vs Финансист» (30 сравнений), «Находки» (17 финансовых решений). Единый поиск по всему контенту.
- **QBI** — страница о компании: кейсы, экспертиза, основатель, контакты, CTA на консультацию.
- **Услуги** — 5 продуктов с ценами и описанием.

## Запуск

Файл самодостаточен. Для локального просмотра откройте `index.html` в браузере.

**Важно про API.** Запросы идут на `https://api.anthropic.com/v1/messages` без ключа — это работает только внутри среды Claude Artifacts, где авторизация подставляется автоматически. Для самостоятельного хостинга (GitHub Pages и т.д.) нужно:

1. Поднять простой бэкенд-прокси (например, Cloudflare Worker), который хранит `ANTHROPIC_API_KEY` и пересылает запросы к API.
2. Заменить URL в функции `callClaude()` на адрес своего прокси.
3. Никогда не вставлять API-ключ в клиентский код.

## Технические заметки

- Модель: `claude-sonnet-4-6`, системные промпты — в константах `SYSTEM_BASE`, `SYSTEM_GENERAL`, `SYSTEM_TAX2026` (последний — готовый, но не подключённый к UI сценарий по Налоговому кодексу РК-2026; вызывается `startChat('tax2026')`).
- Служебные маркеры в ответах модели (`[LEVEL_DONE: ...]`, `[PROFILE_READY: ...]`) управляют прогресс-баром, диаграммой и формой заявки. Для продакшена рекомендуется заменить парсинг текста на tool calling.
- Голосовой ввод — Web Speech API (`ru-RU`); работает в Chrome/Edge/Safari, требует HTTPS.
- Онбординг показывается при каждой загрузке: в прототипе нет персистентности (браузерные хранилища недоступны в среде Artifacts). При переносе на свой хостинг добавьте `localStorage`-флаг или куку.
- Лид-форма в финале диалога сейчас только показывает подтверждение на экране. Точка интеграции с CRM — обработчик `lf-submit` в `addLeadForm()`.

## Известные ограничения прототипа

- Нет сохранения диалога между перезагрузками страницы.
- Поиск — простое подстрочное совпадение, без морфологии.
- Один HTML-файл выбран сознательно для простоты итераций; при росте проекта стоит разнести на модули и собрать через Vite.
